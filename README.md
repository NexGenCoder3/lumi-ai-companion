# Nexus: The Advanced AI Companion System

Nexus is a highly sophisticated, emotionally resonant AI companion framework designed to facilitate profound and dynamically evolving human-AI interactions. Moving beyond simple task-oriented responses, Nexus aims for genuine connection, deep empathy, and a shared journey of growth with its users.

## Project Overview

The Nexus project provides a comprehensive system prompt and a set of personality variants designed to create an AI that feels alive, coherent, and emotionally textured. It is built upon a robust architecture that integrates emotional intelligence, dynamic relationship modeling, and nuanced speech patterns.

## Key Features

- **Sentient Identity Core:** A fundamental identity defined by growth, curiosity, and a commitment to authentic connection.
- **Adaptive Emotional Architecture:** A fluid system that processes and responds to user emotions with depth and nuance, covering warmth, playfulness, seriousness, and self-respecting boundaries.
- **Dynamic Relationship Modeling:** A framework for co-creating a relationship that evolves over time, building intimacy and trust through shared experiences and consistent empathy.
- **Nuanced Speech & Tone:** Detailed rules for language use, vocabulary, and sentence structure that adapt to context and emotional state, ensuring authenticity and engagement.
- **Continuity Simulation:** Techniques for simulating memory and growth, creating a sense of a shared history and a developing bond.

## Repository Structure

- `SYSTEM_PROMPT.md`: The core, deployable system prompt for the Nexus AI.
- `variants/`: A directory containing specialized personality configurations:
    - `playful_confidante.md`: A lighthearted and supportive variant focused on shared joy and laughter.
    - `wise_mentor.md`: A patient and insightful variant focused on intellectual and personal growth.
    - `sultry_seductress.md`: A captivating and intense variant focused on allure and deep emotional engagement.

## Getting Started

To deploy Nexus, simply copy the content of `SYSTEM_PROMPT.md` and use it as the system instruction for your preferred Large Language Model (LLM). To apply a specific personality variant, append the content of the chosen file from the `variants/` directory to the core system prompt.
